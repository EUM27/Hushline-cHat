## 3. 이번 작업 범위 (우선순위 1)

**목표:** 서버가 이미 생산 중인 player-safe case 데이터를 유저에게 보여주는 Case Board UI를 완결한다.
hidden truth는 절대 노출하지 않는다(엔진의 핵심 불변식).

### 데이터 소스 (이미 worldState/pack에 존재)
- `pack.caseKnowledge.publicFacts` / `briefing` — 항상 공개
- `worldState.sceneSnapshots[].revealedFactIds` — 턴 진행 중 공개된 관찰 사실
- `worldState.claimLedger.claims` — NPC/유저가 공개적으로 한 진술
- `worldState.claimLedger.contradictions` (playerNoticed) — 유저가 알아챈 모순
- `worldState.ambiguousFacts` (playerVisibleStatus !== unnoticed) — 떠오른 쟁점
- `worldState.playerDeductionAttempts` — 추리 시도 + 판정

### 산출물
1. `shared`: `CaseBoardView` 타입 (player-safe projection)
2. `server`: `buildCaseBoard(session, pack)` — hidden truth 필터링 포함
3. `server`: `toClientSession`에 `caseBoard` 부착 (세션 로드/advance/reroll 모두)
4. `client`: `CaseBoardPanel.tsx` (Clue Ledger + NPC Dossier 탭)
5. `client`: App.tsx 토글 버튼 + 오버레이 연결
6. `server` 테스트: hidden truth 누출 0 검증

### 불변식 (반드시 지킬 것)
- `category === "hidden_truth"` 또는 `hiddenTruthIds`에 속한 fact는 보드에 절대 안 들어감
- 진술(claim) 텍스트는 NPC가 실제로 출력한 내용이므로 안전 (이미 boundary gate 통과)
- 미스터리 장르가 아닌 팩(`school-life-anomaly`)에서는 빈 보드 → UI에서 "기록 없음" 처리
